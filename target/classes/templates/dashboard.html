✏️ INSERT / UPDATE / DELETE

SELECT — algorithm picks one node to read from

Round Robin: cycles Node1 → Node2 → Node3 → Node1… in strict order regardless of load.
Try: SELECT * FROM employees  ·  SELECT * FROM employees WHERE dept='Engineering'  ·  SELECT dept, COUNT(*) n FROM employees GROUP BY dept

WRITE — algorithm picks primary target node, then replicates to the other 2

Round Robin: the write cycles — 1st write targets Node1 first, 2nd write targets Node2 first, 3rd targets Node3 first, then repeats. The chosen node is the PRIMARY TARGET. The other 2 receive the same SQL as replicas in order.
⚡ How writes work: The algorithm you pick decides which database node gets the write first. That node is the primary target. The other 2 nodes receive the same SQL as replicas — in sequence, each holding an open transaction. If all 3 succeed → all commit together. If any node fails (after up to 3 retries) → all 3 roll back, nothing is saved anywhere.
